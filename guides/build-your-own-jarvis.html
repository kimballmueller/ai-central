Build Your Own Jarvis

"Good evening, sir." Here's the compiled list of everything in the daily-brief demo — every MCP, every Claude Code feature, and the link to set each one up. Wire these together and your whole business reports to you the moment you sit down to work.

⏱ A weekend to wire up 🤖 Claude Code or Claude Desktop 🔌 ~8 connectors

The demo you just watched isn't a single app — it's Claude Code with a stack of MCP connectors (the plugs that let Claude reach your real tools), a couple of built-in features, a set of custom sub-agents, and one scheduled routine that assembles the brief for you every morning. Below is each piece in the order it appeared, what it does, and where to get it.

🎯 Start small. You don't need all of this on day one. Add one connector, confirm Claude can read from it, then add the next. The magic isn't any single MCP — it's that your main agent can pull from all of them at once and write you a single brief.

1 One-shot the dashboard from a reference image

"Give Claude a reference image of what you want, and it'll pretty much one-shot it." Screenshot a dashboard you like — or sketch one — drop it into Claude Code, and ask it to build the UI to match. No connector needed; this is just Claude reading an image and writing the front-end. The whole dashboard in the demo took about five minutes.

🖼️ Paste the image straight into the prompt and say "build this as a single-page dashboard, wire the numbers to live data later." Iterate by screenshotting the result and pasting it back with notes.

2 Voice mode — talk instead of type

Talking to your agent instead of typing is a big productivity boost. Use your OS dictation or a voice front-end so you can fire off instructions hands-free while you're doing other things.

3 ElevenLabs — the British Jarvis voice

Want the replies spoken back in that calm British-butler voice? Connect ElevenLabs for text-to-speech and you've officially become Tony Stark. Pick a voice, grab an API key, and pipe Claude's responses through it.

→ elevenlabs.io

4 Claude for Chrome — give it your browser

"Clouding Chrome to give it access to your browser." With the Claude for Chrome extension, your agent can actually see and act in web pages — pull up dashboards, read stats, click through tools that don't have an API.

→ Claude for Chrome

5 RevenueCat MCP — app revenue

"Over the last 7 days, we had 2,459 new downloads and $4,289 in revenue." That line came straight from the RevenueCat MCP — the official server RevenueCat shipped in 2025, with 26 tools that let Claude read your revenue and even configure products, entitlements, and paywalls by conversation.

→ RevenueCat MCP

6 Buffer MCP — publish everywhere at once

"I'm still posting 3 short-form videos a day." The Buffer MCP lets your agent draft, schedule, and publish to all your social platforms from inside a Claude conversation, so the content engine keeps running without you touching each app. (It runs on Buffer's new API — currently in beta, so it creates and schedules posts; engagement metrics aren't exposed yet.)

→ Post to Buffer from Claude

7 Meta Ads MCP — run campaigns

"You spent $475 at a 1.5% return on ad spend." Meta launched its official Ads AI Connectors (an MCP server, free, in open beta as of April 2026) so Claude can read performance and spin up more campaigns in plain language — letting you give Mark more of your money, faster. No developer app or API token required; it authenticates through Meta Business login.

→ Meta Ads AI Connectors (MCP)

8 Gmail MCP + an FAQ file = 90% of support

"You received 16 customer emails. I resolved 13 automatically." That's the Gmail MCP giving Claude read/write access to your inbox. The trick to the auto-resolution: organize all your FAQs in a markdown file and point the agent at it. Claude handles roughly 90% of customer service — common questions and account issues — and escalates the rest to you.

📝 The FAQ file is the secret. A plain faqs.md with your top 20–30 questions and the approved answers turns the inbox connector into a real support agent. Update the file, and your support gets smarter — no retraining.

→ Gmail MCP (official Google server)

9 Custom sub-agents — Tom, Scout & the team

"I handed it to Tom, the developer agent." This is the part that turns a chatbot into a company. You define custom sub-agents, equip each with the right skills and connectors, and your main agent hands off work to whichever specialist fits the job:

  • Tom — the developer agent. Gets the coding task and the backend PR.
  • Scout — research agent. Tasked with finding the next organic content angles to test.
  • …and any other role you need — a support agent, an ads analyst, a bookkeeper.

Each sub-agent runs in its own context with only the tools it needs, so handoffs stay clean and focused.

→ Claude Code sub-agents · Skills

10 Save it as a routine — the daily brief

The final move: take the entire brief — revenue, content, ads, support, recommendations — and set it as a scheduled routine. Claude assembles it on a cron schedule and has it waiting for you by the time you sit down to work. "Good evening, sir" becomes automatic.

⏰ Write the brief as a single prompt that pulls from every connector above, then schedule it to run each morning. The output is your standup: the numbers, what got handled automatically, and a short list of what actually needs you.
