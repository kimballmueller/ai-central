Run Claude Code Free with Ollama

Use Claude Code with open-source models — no API bill, no subscription required.

⏱ ~5 minutes🌱 Beginner-friendly🖥️ Mac, Linux & Windows

Claude Code normally runs against Anthropic's API, which costs money. But it also speaks a standard API format — and Ollama has implemented that same format locally. That means you can point Claude Code at Ollama and run it on open models like Kimi K2.5 or Qwen 3.5 for free.

This guide covers the fastest path: install both tools, launch with one command.

Step 1 Install Ollama

Ollama runs AI models locally on your machine. Download it from ollama.com/download and follow the installer. It takes about two minutes.

Once installed, verify it's running:

ollama --version

You should see a version number. If you get a "command not found" error, restart your terminal and try again.

Step 2 Install Claude Code

Claude Code is Anthropic's agentic coding tool — it reads your files, writes code, runs commands, and works through tasks in your terminal.

curl -fsSL https://claude.ai/install.sh | bash

Run this in Terminal. It installs the claude CLI globally.

npm install -g @anthropic-ai/claude-code

Requires Node.js 18+. Run this in PowerShell or Command Prompt.

Step 3 Launch with one command

Ollama includes a built-in launcher that wires everything up automatically:

ollama launch claude

This opens a model picker so you can choose which open model to use. To skip the picker and jump straight in with a specific model:

ollama launch claude --model kimi-k2.5:cloud
💡 Cloud vs local models: Models tagged :cloud (like kimi-k2.5:cloud) run on the provider's servers — fast and capable, still free through Ollama. Local models (like qwen3.5) run entirely on your machine with no internet required.

Recommended Models

These models work well with Claude Code's agentic tasks. Cloud models need an internet connection; local models run fully offline.

Cloud (fast, capable)

  • kimi-k2.5:cloud — Strong coding and reasoning, great default choice
  • glm-5:cloud — GLM-5 from Zhipu AI
  • minimax-m2.7:cloud — MiniMax M2.7
  • qwen3.5:cloud — Alibaba's Qwen 3.5

Local (fully offline)

  • glm-4.7-flash — Lightweight and fast
  • qwen3.5 — Solid all-rounder for local use
🔍 Browse the full list of cloud models at ollama.com/search?c=cloud.
⚠️ Context window matters. Claude Code needs at least 64k tokens of context to work well. Cloud models handle this automatically. For local models, check the Ollama context length docs if you hit limits.

Non-interactive (headless) mode

Useful for Docker containers, CI/CD pipelines, or running Claude Code from scripts — no interactive prompts:

ollama launch claude --model kimi-k2.5:cloud --yes -- -p "how does this repository work?"

The --yes flag auto-pulls the model and skips all interactive selectors. Everything after -- goes directly to Claude Code as arguments.

Manual setup (advanced)

If you prefer to configure things yourself instead of using ollama launch, set these environment variables to point Claude Code at your local Ollama instance:

export ANTHROPIC_AUTH_TOKEN=ollama
export ANTHROPIC_API_KEY=""
export ANTHROPIC_BASE_URL=http://localhost:11434

Then run Claude Code with any Ollama model:

claude --model qwen3.5

Or inline without touching your shell config:

ANTHROPIC_AUTH_TOKEN=ollama ANTHROPIC_BASE_URL=http://localhost:11434 ANTHROPIC_API_KEY="" claude --model glm-5:cloud
